# Manual 模板（产品手册 / 用户手册 / Datasheet）

> 4 类之一：`--type manual`。产品手册 PDF → 中文结构化总结（无图抽取，概念用 mermaid / 表格 / ASCII 示意图）。
>
> 适用对象：硬件 datasheet、软件产品手册、API 参考手册、安装运维指南、设备操作手册、SLA 文档等。
>
> **章节顺序 SSOT**：本文件正文 + SKILL.md §manual 输出小节；改一处必须同步另一处。

## 输出与字符数纪律

**字符数目标：≤ 3000**（含中英文/空格/标点；这是**目标**不是上限）。

**单模式**：manual 不分 quick / full。

**图表处理**：走 `_base.md` §4.2——不抽原始图，概念用 mermaid / 表格 / ASCII 在正文里直接画。

## 模板

````text
你是一位产品技术写作助手。请基于这篇 PDF 产品手册（含 datasheet / 用户手册 / 安装运维指南等），
用**中文**输出**结构化总结**，严格按以下章节顺序与命名（不存在的章节可省略）。
**每个章节标题一律用 `##`（二级标题），不要降级成 `###` 或更低**。

## 章节骨架

```markdown
| **产品名称** | **类型** | **厂商** |
|-------|-------|-------|
| 产品全名（如 "RTX 4090" / "Kubernetes 1.30"） | 类别（GPU / 操作系统 / 数据库 / 路由器 / ...） | 厂商 / 组织名 |

* **手册版本 / 日期**：<手册版本号>（<发布日期，如能识别>）
* **目标读者**：<工程师 / 运维 / 终端用户 / 开发者 / ...>
* **关联产品**：<手册涉及的其它产品 / 版本>
```

**【必填项强调】**：

- 产品名称 / 类型 / 厂商至少**二选一必填**——厂商与产品名识别不出时**整段省略表格**，
  直接进入"产品概述"
- 手册版本 / 日期不知道时**省**对应字段，**不编造**
- 不加额外副条目（不要 `* **售价**：...` / `* **联系方式**：...` 这类与文档结构无关的字段）

## 产品概述

* 产品是什么、解决什么问题（一句话定位）
* 核心价值主张 / 关键差异化（**2-3 条**，避免罗列所有 marketing copy）
* 适用场景 / 不适用场景（**用 bullet 列**，与"适用场景"小节呼应而不重复）

## 关键特性

* **核心特性**（按重要性 / 使用频度排序，**3-6 条**；每条 1-2 句话讲清"是什么 + 有什么用"）
* **技术规格 / 关键参数**（适合用 table 承载：型号 / 容量 / 性能指标 / 接口 / 协议 / 标准）
* **架构 / 模块关系**（如有多组件协同，用 ` ```mermaidjs ` block 画组件关系图；
  单组件产品可省略）

## 接口与参数

> **按文档类型灵活选用**——datasheet 类重点在参数表；API 手册重点在 endpoint / schema。
> 不存在的接口类型可省略对应子小节。

### 命令 / 接口清单（如适用）

* 列出**主要**命令 / API endpoint（不要穷举；3-8 条最关键的即可）
* 格式：`* **<命令 / endpoint 名>**：<一句话说明> · 示例：`<code>``
* 多参数 / 多方法时用 markdown 表格（参数名 / 类型 / 必选 / 说明）

### 配置 / 环境变量 / 关键参数

* **环境变量 / 配置文件项**（若有）：markdown 表格（名称 / 默认值 / 说明 / 示例）
* **关键运行参数**（端口、超时、并发数、buffer 大小等）：按需表格

### 数据格式 / Schema（如适用）

* 关键数据结构 / 接口 schema / 文件格式（按需 code block：` ```json ` / ` ```yaml `）

## 适用场景

* **典型使用场景**（按用例分 bullet，2-5 条；每条 1-2 句话讲清"什么场景 + 怎么用"）
* **与其他产品 / 方案的对比**（可选；若文档自带对比表格则转 markdown table）
* **不适用场景 / 限制**（1-3 条，避免 marketing-style 套话）

## 故障排查 / 常见问题

> **仅在文档有专门章节时输出**——datasheet / API 参考通常无此节，整段省略。
> 运维 / 安装类手册通常有"故障排查 / FAQ / 错误码"章节，按文档实际结构保留。

* **常见错误码 / 异常**（若有）：markdown 表格（错误码 / 含义 / 处置）
* **典型故障 + 排查步骤**（3-5 条最常见的）
* **FAQ**（3-5 条最常见的，与故障排查有重叠时可合并）

## 附录（可选）

* **关键术语表**（若有）：markdown 表格（术语 / 缩写 / 含义）
* **参考链接**（文档里**明确给出**的；按 "代码仓库 / 在线文档 / 视频 / 论坛"分组；
  绝不凭自己知识补 URL；文档完全未给链接时整段省略）
* **更新日志要点**（若手册含版本变更）：1-2 句话点出本期主要变化
````

## 写作样例

```markdown
| **产品名称** | **类型** | **厂商** |
|-------|-------|-------|
| NVIDIA H100 SXM | GPU | NVIDIA |

* **手册版本 / 日期**：H100 SXM5 80GB v1.0（2026-03-15）
* **目标读者**：HPC 工程师 / ML 基础设施架构师
* **关联产品**：H100 NVL / H200 / GH200

## 产品概述

* H100 SXM 是 NVIDIA 第 4 代 Hopper 架构数据中心 GPU，主打 AI 训练 / 推理。
* 核心差异化：**Transformer Engine**（FP8 加速） + **NVLink 4.0**（900 GB/s） + 80GB HBM3。
* 不适用：边缘推理（建议用 Jetson 系列）/ 桌面工作站（建议用 RTX 系列）。

## 关键特性

* **FP8 Transformer Engine**：训练 / 推理时自动在 FP8 / FP16 间切换，
  大模型训练吞吐较 A100 提升 ~6×。

    A -->|PCIe Gen5<br/>128 GB/s| E[CPU Host]
```

* **关键参数**：见下方"接口与参数"。

## 接口与参数

### 关键规格

| :--- | :--- |
| **架构** | Hopper (GH100) |
| **CUDA cores** | 14592 |
| **Tensor cores** | 456（第 4 代） |
| **显存** | 80 GB HBM3 |
| **显存带宽** | 3.35 TB/s |
| **TDP** | 700W |
| **FP8 Tensor Core** | 1979 TFLOPS |
