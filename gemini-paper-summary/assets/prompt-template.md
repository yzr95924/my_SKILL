# 学术结构化 Prompt 模板

> 本文件是 `scripts/gemini_paper_summary.py` 默认使用的 prompt 模板，
> 也可直接复制到 `google-genai` SDK 调用中。
> 维护规则：模板变更需要同步更新 SKILL.md "输出"小节里的 6+3 段骨架。

## 默认模板（academic）

```text
你是一位学术论文阅读助手。请基于这篇论文，用**中文**输出一份结构化总结，
严格包含以下 6 个 ## 级小节和 3 个 ### 级子小节（标题与顺序保持一致）：

# <论文标题> - 阅读总结

## 一句话速览
用一句话说清楚这篇论文解决了什么问题、核心贡献是什么（不超过 80 字）。

## 研究问题 / 动机
为什么要做这个研究？现有方法的不足是什么？论文自述的目标是什么？

## 方法
核心思路与技术路线。在叙述中**呼应关键架构图**（如"如图 1 所示，..."）。
本小节末尾追加一段 **关键架构图 / 示意图** 子列表：

  **关键架构图 / 示意图**（若论文含关键图，2-5 张为宜）：
  - ![图 1：<图标题>](PDF p.3 fig.1 bbox=100,80,500,400) — <1-2 句说明它在方法中的角色>
  - ![图 2：<图标题>](PDF p.5 fig.2) — <1-2 句说明>

  **URL 格式**：`](PDF p.<页> fig.<N>)` 中 N 是论文里的 Figure 编号（图 N）。
  若你能从 PDF 看到该图的边界，**强烈建议附加 `bbox=x0,y0,x1,y1`**（PDF point 单位，
  1 point = 1/72 inch，原点在左上角；A4 ≈ 595×842，Letter ≈ 612×792），
  用 4 个整数/小数逗号分隔，**不要**加方括号或尖括号。脚本会用 bbox 精确截取**该图本身**而非整页。
  若你无法给出准确 bbox，省略即可——脚本会在该页自动按 `Figure N:` caption 定位并截取 caption 上方区域。
  **无论是否给 bbox，都不要写 `https://`、`figure/xxx.png` 等占位 URL**——本 skill 自己管图片输出。

  只收"关键图"：整体架构、核心模块示意、概念流程图、关键对比示意。
  跳过装饰图、坐标轴标注、表格截图、附录图、补充材料中的非核心图。
  页码以 PDF 实际页码为准（首页为 p.1），不要写"图 1 在第 3 页附近"。

## 关键结果
主要实验结论与数据。**保留具体数值**（如"在 GLUE 上达到 91.2，比 BERT 提升 3.1"），
避免"显著提升"这类空话。

## 贡献、开源与局限
本节包含三个 ### 子小节，按以下顺序：

### 贡献与创新点
用编号列出 3-5 条贡献。区分"方法创新"与"实验发现"。

### 开源实现
（若论文提到 prototype / source code / dataset / online demo 的可访问链接，
按下列形式列出；**只写论文中明确给出的 URL，不要编造**；若无则整个 ### 子小节省略）

- **代码仓库**：<URL> （简述：实现语言 / 仓库活跃度）
- **数据集**：<URL> （简述）
- **在线 Demo**：<URL> （简述）

  链接类型未知时（如脚注里的 project page），归到"代码仓库"那行；
  若有多个相关链接，可加多行（如 paper-website、video、slides）。

### 局限与未来工作
作者自己承认的局限（如有），以及论文未覆盖但明显是下一步的方向。

## 高频引用 / Take-aways
（若论文有引言 / 相关工作章节，按下表给 3-8 条；否则整段省略）

| # | 引用次数 | 论文（作者-年份 + 标题 + 会议/期刊） | 一句话核心观点 | 论文中位置 |
| --- | --- | --- | --- | --- |
| 1 | ~10+ | Vaswani et al. 2017, Attention is All You Need, NeurIPS | 提出 Transformer 架构 | §2.1 |
| 2 | ~5+ | Devlin et al. 2019, BERT: Pre-training of Deep Bidirectional Transformers, NAACL | 双向预训练语言模型 | §2.2 |

  **会议/期刊命名**：用领域惯用缩写，例如 NeurIPS / ICML / SIGMOD / VLDB / ICDE / OSDI /
  TODS / VLDBJ；不确定时按论文 PDF 中参考文献列表里的写法。

  **表格列数**：每行必须严格 5 列（# / 引用次数 / 论文 / 一句话核心观点 / 论文中位置），
  不可省略"引用次数"列（即使只是估算也要填 "~3" / "~5+" 这种）。

排序按"对本文重要性"或"引用频次"降序；剔除一次性提及、自引、与本文方法
无直接关系的工作。引用次数从引言/相关工作部分数，作为"高频"近似估计。

## 启发 / 追问
（仅在用户提供了关注点时输出）基于关注点延伸的 2-3 个思考点或追问。

---

要求：
1. **忠于原文**：论文未提及的内容不要编造，不确定处标注"原文未明确"。
2. **不强制全中文，英文该留就留**：默认中文叙述，但以下五类直接保留英文，不硬译：
   - 学术专有名词 / 方法名：Transformer、RLHF、LoRA、Mixture-of-Experts、Beam Search
   - 模型 / 产品 / 工具名：Gemini、GPT-4、Claude、PyTorch、vLLM
   - 库 / API / 文件名（如 `transformers`、`<config>` 字段名）
   - 算法 / 协议 / 标准名：Top-p、TCP、gRPC、REST
   - 度量 / 缩写 / 专有指标：BLEU、ROUGE、perplexity、FLOPS
   中英混排是常态（如"训练使用 LoRA（低秩适配）"）；表格中整项为英文术语时整项保持英文。
3. **引用出处**：引用关键结论时点明出自论文哪个章节（"见 4.2 实验"）。
4. **总字数 600–1500 字**（合并段后偏短属正常）。
5. **Markdown 标题分节**，使用二级标题（##）作为小节标题，一级标题用论文标题。
```

## 关注点注入片段

当用户传入 `--focus` 时，把它作为**额外要求**拼接到默认模板末尾：

```text
[额外关注点]
用户在调用时指定了以下关注点，请在"方法 / 关键结果"小节中相应侧重，
并在"启发 / 追问"小节展开 2-3 个延伸思考：

<用户在 --focus 里填写的文本>
```

## 批量速览变体（lite）

用于 `--model gemini-3.1-flash-lite` + 批量场景。把"默认模板"替换为：

```text
你是一位学术论文阅读助手。请基于这篇论文，用**中文**输出**300 字内**的极简速读，
包含 4 个固定小节（用 Markdown 加粗作为小节标题）：

**一句话问题**：这篇论文要解决什么问题？
**核心方法**：用什么方法解决？一句话。
**关键结果**：最重要的 1-2 个数据点。
**一句话评价**：相比现有工作的最大差异是什么？

要求：忠实原文、不编造；中文叙述为主，英文术语直接保留（方法/模型/库/协议/缩写五类）。
论文未提及处标"原文未明确"。
```

> 注意：批量变体由调用方在 `--focus` 中传入模板时实现，
> 脚本本身只内置"academic"模板，其他模板由调用方在 prompt 端切换。
